# 照见一念（Glimmer）墨刀 AI 提示词文档（Web 优先，多端扩展）

- 适用产品：照见一念（Glimmer）
- 适用工具：墨刀 AI
- 设计策略：跨平台多端产品，先以 Web 端原型提示词为主，再向移动端 App / 平板 / 桌面端扩展
- 目标：为产品原型生成提供可直接使用的高质量提示词

---

## 1. 产品与设计背景

照见一念（Glimmer）是一款帮助用户在不确定情境下获得启发、完成反思、形成微行动的轻量心理工具。产品核心链路为：

```text
输入问题
→ 稳定入口（可选）
→ 情绪识别（可选）
→ 自由倾诉整理（可选）
→ 问题整理确认（可选）
→ 模式选择（可选）
→ 启发式答案
→ 思考卡片
→ 深入回答
→ 总结与微实验
→ 保存为情绪卡 / 回看 / 回访补记 / 恢复未完成会话 / 洞察
```

产品不是聊天机器人，也不是占卜应用，而是一个围绕“启发、反思、行动”的结构化心理工具。

设计气质要求：

- 温和
- 克制
- 清明
- 有留白
- 有微光感
- 适合深度阅读与截图分享
- 不玄学，不压迫，不说教

---

## 2. 多端设计策略

## 2.1 终端策略

本产品为跨平台多端应用，建议按以下顺序推进：

1. **Web 端优先**
   - 先产出桌面 Web 与响应式 Web 原型
   - 适合产品定义、研发沟通、快速验证
2. **移动端适配**
   - 基于 Web 结构压缩为移动端体验
   - 保持主链路不变，强化单手操作
3. **平板 / 桌面增强**
   - 扩展留白、双栏布局、洞察页可视化

## 2.2 当前提示词重点

本提示词文档以 **Web 端页面原型** 为主，适合：

- 桌面浏览器场景
- 宽屏响应式布局
- SaaS / Web App 风格交互
- 后续再迁移到 App

## 2.3 Web 端设计原则

- 页面采用居中主内容区
- 支持大面积留白与沉静氛围
- 首页、结果页、洞察页更适合双栏或中轴布局
- 表达以卡片、模块、分层信息为主
- 避免做成传统工作台后台

## 4. 页面级提示词（Web 端优先）

## 4.1 首页 / 提问页 Home / Ask Page

### 页面目标

让用户低门槛进入产品，并写下一个真实问题。

### 页面结构建议

Web 端建议使用：

- 顶部简洁导航
- 中央主内容区
- 中轴布局或左文案右输入双栏布局

### 墨刀 AI 提示词

设计“照见一念（Glimmer）”的 Web 端首页 / 提问页，桌面端优先，整体为深色高级界面，具有温和、克制、清明的气质。页面核心目标是鼓励用户写下一个当前最想问的问题。顶部为极简导航栏，包含品牌名、历史记录入口、洞察入口或设置入口。页面主体使用居中双栏布局：左侧为品牌文案区，展示一句有启发感的标题和简短说明；右侧为核心提问区，包含大面积多行输入框、示例问题标签、主按钮“开始照见”。输入框要有强视觉焦点，圆角大、边界柔和、有轻微发光感。示例问题以标签 chips 形式展示，例如“我要不要离开现在的工作？”“为什么我总是下不了决心？”。整体不应像搜索引擎，也不应像聊天窗口，而更像一个可以安静写下念头的空间。

- 左文案右输入

- 高级深色产品官网感与 Web App 感结合

### 页面结构建议

- 居中单卡布局
- 顶部保留轻量进度感
- 中部为情绪标签选择器

### 墨刀 AI 提示词

## 4.2A 稳定入口页 Soothing Gate

### 页面目标

让用户在还无法组织语言时，先获得一个 20–60 秒的安静过渡，再决定是否继续提问。

### 页面结构建议

- 居中单卡或中轴布局
- 中间为呼吸节律或微光动画
- 下方提供继续提问、先随便说说、稍后再来

### 墨刀 AI 提示词

设计“照见一念（Glimmer）”的 Web 端稳定入口页，整体采用深色微光界面和中轴布局，气质温和、克制、留白充足。页面用于用户在高情绪张力下先缓一缓，而不是立刻回答问题。页面中央展示一个柔和的呼吸节律视觉，如发光圆环、渐变呼吸点或缓慢扩散的光晕。标题可以是“如果你现在很乱，先不用急着想清楚”，副标题强调“先让这一口气慢一点，再看看要不要继续”。底部提供三个动作按钮：“继续提问”“先随便说说”“稍后再来”。整体不应像冥想 App，也不应像治疗工具，而应像产品主流程前的温柔缓冲层。

### 高保真视觉版提示词

为“照见一念（Glimmer）”设计一个 Web 端稳定入口页高保真原型，桌面端优先，页面定位是用户在高情绪张力下进入正式提问前的温柔缓冲层。整体视觉语言为“东方微光 + 现代结构”，气质安静、克制、清明、留白充足，带轻微仪式感，但不能像冥想 App、心理治疗工具或睡眠应用。页面采用深色高级界面，背景以深墨蓝、雾灰黑、青灰蓝渐变为主，叠加极轻的雾面颗粒、柔和光晕和低对比流动光层，营造安静包裹感。主内容区使用中轴居中构图，视觉焦点集中在页面中央的“呼吸节律装置”，可表现为一枚缓慢呼吸的发光圆环、半透明光球、渐变扩散波纹或带雾感的微光核心，动画感需柔和、缓慢、有节律，不夸张、不炫技。

页面标题建议呈现为“如果你现在很乱，先不用急着想清楚”，采用偏细但有气质的中文字体风格，字重中等，字间距舒展，保持高级感和呼吸感。副标题放在标题下方，内容为“先让这一口气慢一点，再看看要不要继续”，行宽控制克制，颜色略浅于正文但不要过亮。整体文案排版要显得安静、可信、不说教。中央呼吸视觉周围可加入极轻的时间提示或状态暗示，例如“20 秒安静一下”，但应弱化处理，不能破坏沉浸感。

页面下方设置三个操作按钮，形成清晰主次层级。主按钮为“继续提问”，采用带微光边界的实心按钮，具有明确聚焦感；次按钮为“先随便说说”，适合做成半透明描边按钮或次高权重按钮；弱操作为“稍后再来”，使用低对比文字按钮。按钮需要有完整的悬停、按下、聚焦状态，桌面端交互反馈要细腻，例如轻微发光增强、边框亮度变化、背景雾化提升。整体按钮区不应像表单提交区，更像用户被温柔引导继续往下走的分流入口。

页面可在顶部保留极简品牌识别，如小尺寸品牌名、淡化导航或极轻的返回入口，但不要出现复杂导航栏，不要让页面像 SaaS 控制台。页面四周应有大量留白，内容密度低，强调“先停一下”的情绪空间。材质上建议使用雾面玻璃、低对比渐变、轻发光描边、柔和阴影和大圆角卡片语言，但必须克制，避免赛博霓虹感、过度紫色、过亮高光和宗教化意象。整体目标是让用户一进入页面，就感到被接住、被放慢、被允许先不回答，而不是被要求立刻表达。

### 关键词补充

- Web 稳定入口
- 中轴呼吸动画
- 微光节律
- 情绪缓冲层

## 4.2B 自由倾诉整理页 Unload Page

### 页面目标

允许用户先输入碎片化想法或语音倾诉，再由系统帮助整理成正式问题。

### 页面结构建议

- 居中单栏布局
- 上方为容纳式文案
- 中部为大输入区或语音转写区
- 下方为 AI 提炼结果卡与继续按钮

### 墨刀 AI 提示词

设计“照见一念（Glimmer）”的 Web 端自由倾诉整理页，采用深色微光界面、居中单栏布局。页面用于用户还说不清问题时，先把脑中碎片表达出来。顶部放一条温和文案，例如“不需要说完整，你可以先把脑子里的那一团说出来”。中间是一块大面积输入区，视觉上像一张可倾诉的卡片，而不是聊天输入框。页面支持语音转写入口。输入区下方展示一张“帮你整理出来的问题”结果卡，让用户看到系统如何从碎片化表达中提炼问题核心。底部按钮为“就从这个问题开始”与“再整理一下”。整体应有被接住的感觉，但不能做成客服对话或情感陪聊界面。

### 高保真视觉版提示词

为“照见一念（Glimmer）”设计一个 Web 端自由倾诉整理页高保真原型，桌面端优先，页面定位是用户在还无法清楚表达问题时，先被允许低压力地说出来，再由系统帮助整理。整体视觉语言延续“东方微光 + 现代结构”，但相比首页更柔软、更有容纳感。页面采用深色高级界面，背景使用深墨蓝、烟灰黑和极淡青灰光晕渐变，并叠加非常轻的雾面纹理，让页面有安静包裹感。布局以居中单栏为主，内容区宽度适中，留白充足，避免像表单页、客服聊天页或笔记软件。

页面顶部放置一段容纳式引导文案，如“不需要说完整，你可以先把脑子里的那一团说出来”。标题与说明需要呈现可信、温和、不过度心理化的语气，字体偏细，行距舒展。中部主视觉是一张高权重的大输入卡，像一张可以安心倾诉的发光书写卡片，材质可采用深色玻璃拟态、雾面面板、柔和描边和低亮度内发光。输入区应明显大于普通输入框，更像“承接容器”，支持多行输入和语音转写入口。语音入口可设计为悬浮麦克风按钮、带波纹的轻交互图标或卡片侧边的功能按钮，但不能做成聊天录音界面。

输入区下方展示一张“帮你整理出来的问题”结果卡，作为第二视觉层级。结果卡要与输入区形成清晰对比，体现“混乱被轻轻整理”的感觉。卡片可用更明一些的边界、柔和微光和主问题高亮排版，突出系统提炼出的核心问题。若有补充说明或整理依据，应以轻量小字或标签形式呈现，避免像 AI 解释面板。页面底部设置两个主要动作按钮：“就从这个问题开始”和“再整理一下”，前者为主按钮，强调确定和继续，后者为次按钮，强调保留选择权。整体交互反馈应细腻克制，按钮悬停时可有轻微发光增强、阴影变化和透明度过渡。

页面整体氛围应让用户感到“这里可以先乱一点，也没关系”。视觉上建议加入少量微光线条、低对比渐变和模糊背景层，增强情绪承接感，但必须避免做成陪聊产品、树洞社区、冥想工具或情绪热线界面。最终效果应像一个高保真、可信、安静的心理整理入口。

### 关键词补充

- Web 自由倾诉页
- 单栏沉静布局
- 倾诉卡片
- AI 问题整理

## 4.2C 问题整理确认页 Question Refine Page

### 页面目标

让用户确认系统提炼出来的主问题是否准确，并从 1–2 个侧重点里选择更适合继续的一条。

### 页面结构建议

- 居中单栏布局
- 上方为原始表达摘要
- 中部为主问题卡与可选侧重点
- 下方为去向按钮：继续反思 / 先缓一缓 / 先不决定

### 墨刀 AI 提示词

设计“照见一念（Glimmer）”的 Web 端问题整理确认页，采用深色微光界面与居中单栏布局。页面用于用户在自由倾诉之后确认“这是不是我真正想问的问题”。顶部展示一行轻量摘要，回显用户刚刚表达的碎片内容，但不要过长。页面中部是一张高权重主问题卡，清楚展示系统提炼出的核心问题，例如“我要不要因为持续疲惫而重新评估当前工作？”。主问题卡下方展示 1 到 2 个可选侧重点，以轻卡片或按钮形式呈现，例如“我是真的想离开，还是只是太累了？”、“我现在最需要恢复状态，还是评估机会？”。页面底部提供三个动作按钮：“就从这个问题开始”“先缓一缓”“先不决定”。整体要让用户感到内容已经从混乱进入了一点秩序，但仍保留选择权，不应像审批页面，也不应像问卷分支树。

### 高保真视觉版提示词

为“照见一念（Glimmer）”设计一个 Web 端问题整理确认页高保真原型，桌面端优先，页面定位是让用户在碎片化倾诉后，看到系统帮助自己从混乱进入秩序，并确认“这是不是我真正想问的问题”。整体视觉语言延续产品的深色微光体系，但比自由倾诉页更强调结构感、清晰度和聚焦感。背景以深墨蓝、灰黑、低饱和青灰渐变为主，辅以轻微雾面颗粒和柔和光晕，让页面有安静但更有方向感的气质。整体布局采用居中单栏，信息分层明确，从上到下依次为原始摘要、主问题卡、侧重点选择、行动按钮。

页面顶部展示用户原始表达的轻量摘要，可用小字号、低对比文本或收纳式摘要卡呈现，视觉权重较低，目的是告诉用户“这些碎片被接住了”。页面中部是一张高权重主问题卡，卡片需要明显成为全页视觉中心，采用大圆角、柔和发光边界、略亮于背景的深色面板，以及更大的主问题文字排版。主问题应被呈现得清晰、稳定、可信，像一句终于被说准的话。主问题卡下可附一句简短辅助说明，如“如果这句话接近你的真实困扰，我们就从这里继续”，但文案必须克制。

主问题卡下方展示 1 到 2 个侧重点选项，可用轻卡片、分支按钮或低压选择模块呈现。它们的视觉层级低于主问题，但仍需清晰可点，适合用半透明描边、轻微高亮和 hover 状态区分。用户选中时，应出现明确但温和的视觉反馈，例如边界发光、底色微提亮、图标出现或文字加重。页面底部提供三个去向按钮：“就从这个问题开始”作为主按钮，“先缓一缓”为次按钮，“先不决定”为弱操作文本按钮。整体要体现出选择权仍在用户手里，而不是系统替用户下结论。

视觉上要让用户感到：内容已经从混乱进入一点秩序，但这个秩序不是强迫，而是轻轻被整理出来。避免做成审批确认页、智能表单页、问卷分流树或管理后台流程页。最终效果应像一个高保真、沉静、可信的“问题聚焦界面”，让人自然愿意继续下一步。

### 关键词补充

- Web 问题确认页
