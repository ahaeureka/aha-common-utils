## 1. 本次补充开发目标

在原 MVP 主链路基础上，新增以下增强能力：

1. 情绪识别入口
2. 双模式触发系统
3. 心理视角卡片体系升级
4. 认知偏差识别与提醒
5. 未来自我对话
6. 反向验证卡
7. 微实验行动系统
8. 问题模式与决策风格分析

本次补充重点不是全部同时上线，而是建立：

- 可灰度发布的模块边界
- 可埋点追踪的数据结构
- 可逐步启用的 AI 策略体系

---

## 2. 推荐版本拆分

## 2.1 MVP+（建议优先）

优先实现：

- 情绪识别入口
- 双模式触发系统
- 心理视角卡片体系升级
- 微实验行动系统

原因：

- 开发成本可控
- 对主链路体验提升最明显
- 对转化和完成率影响最大

## 2.2 Beta 增强

后续实现：

- 认知偏差识别与提醒
- 未来自我对话
- 反向验证卡
- 问题模式识别
- 决策风格分析

原因：

- 依赖更多上下文和历史数据
- 需要更严格的输出风格控制
- 更适合在有基础数据后迭代

## 3. 前端补充拆解

## 3.0 新增缓冲层页面：稳定入口、自由倾诉、问题提炼与恢复入口

### 页面目标

- 在正式进入情绪识别与答案生成前，先承接高情绪、表达混乱、尚未成句的问题输入。
- 保持轻量，不延长链路过多，不做深度冥想或长篇陪聊。

### 新增页面

#### `SoothingGate`

- 呼吸 / 落地 / 稍等一下 三种轻稳定方式
- 15 秒 / 30 秒 / 60 秒轻量时长
- 支持“直接继续”跳过

#### `UnloadPage`

- 文本自由倾诉输入
- 可选语音转写按钮
- 一键整理为标准问题
- 返回整理后问题卡片供用户确认继续

#### `QuestionRefinePage`

- 展示主问题提炼结果
- 展示 1–2 个可选侧重点
- 支持继续反思 / 先缓一缓 / 先不决定
- 支持返回修改原始表达

#### `ResumeEntrySheet`

- 在首页或历史页提示可恢复会话
- 展示最近一条未完成会话摘要
- 支持继续整理、继续反思、仅查看结果
- 支持忽略本次恢复提示

### 组件拆解

- `BreathingHalo`
- `DurationSegmentedControl`

- `FreeformTextArea`
- `VoiceTranscribeButton`
- `RefinedQuestionCard`

## 3.1 新增步骤页：情绪识别层 `EmotionGate`

### 功能

- 在用户输入问题后、生成答案前展示轻量情绪选项
- 支持跳过
- 将选择结果写入当前 `session`

### 组件拆解

- `EmotionTagSelector`
- `EmotionTagChip`
- `SkipEmotionButton`

### 数据字段

```ts
interface EmotionState {
  emotionTag?: 'anxious' | 'tired' | 'reluctant' | 'afraid' | 'conflicted' | 'unclear';
  source?: 'user_selected' | 'model_inferred';
}
```

### 交互要求

- 不超过 6–8 个选项
- 支持单选
- 支持“说不清”
- 支持直接跳过

### 验收

- 选择不超过 1 次点击完成
- 跳过不影响主流程
- 选择结果可在后续结果页生效

## 5. 后端接口补充拆解

## 5.0 新增缓冲层与沉淀层接口

### 新增接口

- `POST /api/v1/sessions/{sessionId}/soothing`
- `POST /api/v1/sessions/{sessionId}/unload`
- `POST /api/v1/sessions/{sessionId}/refine-confirm`
- `GET /api/v1/sessions/resumable`
- `POST /api/v1/sessions/{sessionId}/resume`
- `PATCH /api/v1/sessions/{sessionId}/journal-note`
- `PATCH /api/v1/sessions/{sessionId}/followup`
- `GET /api/v1/users/{userId}/journal/weekly-review`

### 后端处理要求

- `soothing` 接口只记录最小必要状态，不引入复杂多媒体资产依赖。
- `unload` 接口要同时保存原始文本和整理后的标准问题，便于后续评估整理质量。
- `refine-confirm` 接口要支持用户确认主问题、选择侧重点或回写自定义问题文本。
- `resumable` 接口需返回最适合恢复的一条会话，以及可用动作，避免前端自行拼装恢复逻辑。
- `resume` 接口需负责把用户带回正确阶段，而不是仅返回 session 详情让前端硬判断。
- `journal-note` 与 `followup` 接口需支持幂等更新，允许历史页多次编辑。
- `weekly-review` 优先读取聚合快照，避免每次临时扫描全量历史。

## 5.0A 新增接口：确认问题提炼结果

### 接口

`POST /api/v1/sessions/{sessionId}/refine-confirm`

### 用途

- 对应 `QuestionRefinePage`
- 把整理后的主问题正式写回 `session.question`
- 记录用户选择的侧重点或“先不决定”分流

### 请求建议

```json
{
  "confirmedQuestionText": "我要不要因为持续疲惫而重新评估当前工作？",
  "selectedFocus": "我是真的想离开，还是只是太累了？",
  "nextAction": "continue"
}
```

### 返回建议

```json
{
  "sessionId": "sess_xxx",
  "status": "draft",
  "question": {
    "text": "我要不要因为持续疲惫而重新评估当前工作？"
  },
  "selectedFocus": "我是真的想离开，还是只是太累了？",
  "nextStep": "emotion_gate"
}
```

### 处理逻辑补充

1. 校验当前会话状态必须允许从 `question_refined` 继续
2. 若用户改写了主问题，保留原提炼结果用于质量评估
3. `nextAction = soothe` 时直接返回 `soothing_gate`
4. `nextAction = decide_later` 时写入可恢复状态并结束本次链路

## 5.0B 新增接口：恢复未完成会话

### 接口

`GET /api/v1/sessions/resumable`

### 用途

```json
      "updatedAt": "2026-03-11T23:41:00Z",
      "availableActions": ["continue_refine", "continue_reflection", "view_result"]
    }
  ]

```

### 配套接口

`POST /api/v1/sessions/{sessionId}/resume`

### 用途

### 返回建议

```json
{
  "sessionId": "sess_xxx",
  "resumeTo": "question_refine",
  "snapshot": {
    "rawText": "我最近一直很累，也不知道是不是该离开现在的工作。",

  }

```

## 6. AI 编排补充拆解

## 6.1 新增服务：`emotionRouterService`

### 作用

- 接收用户显式情绪标签或模型推断情绪
- 为 Trigger / Reflection / Action 选择更贴合的策略模板

### 输出

```json
{
  "emotionTag": "tired",
  "confidence": 0.91,
  "strategyHint": "stabilize_first"
}
```

---

## 6.2 Trigger Service 增强

### 新输入

- `emotionTag`
- `insightMode`

### 新规则

- `reflective` 模式优先输出启发型、情绪承接型语言
- `decisive` 模式优先输出结构型、验证型语言
- 当 `emotionTag = tired/anxious` 时，避免推动性过强措辞

---

## 6.3 Reflection Service 增强

### 新能力

- 基于情绪和问题分类从 6 大心理卡池中选卡
- 在满足条件时追加 `reverse_check` 卡

### 选卡规则建议

- 卡 1：贴合当前情绪
- 卡 2：贴合问题类型
- 卡 3：用于平衡和拓宽视角

### 反向验证卡触发条件建议

- 用户问题出现明显立场倾向
- 强烈使用“是不是就该”“我是不是必须”“肯定没希望了”等表达
- 高情绪、高不可逆决策场景

---

## 6.4 新增服务：`biasDetectorService`

### 输入

- 问题文本
- 用户回答
- 选中卡片类型

### 输出

```json
{
  "biases": [
    {
      "code": "catastrophizing",
      "label": "灾难化倾向",
      "message": "你可能在高估最坏结果的持续时间。"
    }
  ]
}
```

### 规则

- 最多返回 1–2 条
- 仅在高置信度时展示
- 使用提示语气，不用诊断语气

---

## 6.5 新增服务：`futureSelfService`

### 输入

- 问题文本
- 问题分类
- 情绪标签
- 反思总结

### 输出

```json
{
  "role": "三个月后的你",
  "message": "先不要让疲惫替你做长期决定。"
}
```

### 规则

- 角色固定在有限集合中，避免风格失控
- message 长度可控
- 与品牌语气保持一致

---

## 6.6 Action Service 增强

### 新要求

- 普通建议升级为微实验
- 必须返回标签
- 尽可能返回 `ifThenPlan`

### 行动标签推断逻辑

- 收集信息 → `info_collect`
- 澄清标准 → `clarify`
- 调整状态 → `emotion_stabilize`
- 低成本试探 → `small_probe`

---

- 行动采纳历史

### 输出
