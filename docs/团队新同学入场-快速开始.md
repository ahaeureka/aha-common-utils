# 团队新同学入场（快速开始）

目标：在 10 分钟内完成本地环境初始化并验证数据库可用。

适用目录：`/app`

---

## 0. 前置要求

- 已安装：Python 3.12、`uv`、PostgreSQL、`direnv`
- 当前目录：

```bash
cd /app
```

---

## 1. 一次性初始化（推荐）

```bash
make env-prepare
make env-dev
make env-local-init
```

执行后会生成（存在则不覆盖）：

- `.env`
- `.env.development`
- `.env.local`
- `.envrc`

---

## 2. 填写本地敏感变量（不入库）

编辑 `.env.local`，至少填写：

```dotenv
DB_URL_PSQL_DEVELOPMENT=postgresql://<user>:<password>@<host>:5432/<db_dev>
DB_URL_PSQL_TEST=postgresql://<user>:<password>@<host>:5432/<db_test>
DB_URL_PSQL_PRODUCTION=postgresql://<user>:<password>@<host>:5432/<db_prod>
```

> `.env.local`、`.envrc`、`.direnv/` 已加入 `.gitignore`，默认不会提交。

---

## 3. 启用 direnv 自动注入

```bash
direnv allow
make env-local-check
env | grep '^DB_URL_PSQL_'
```

看到环境变量输出即表示自动注入生效。

---

## 4. 安装依赖 + 初始化开发库

```bash
make deps
make db-bootstrap-dev DB_URL_PSQL_DEVELOPMENT="$DB_URL_PSQL_DEVELOPMENT"
```

可选校验：

```bash
make db-current
make db-verify DB_URL_PSQL_DEVELOPMENT="$DB_URL_PSQL_DEVELOPMENT"
```

---

## 5. 常见问题（新同学高频）

- `缺少 DB_URL_PSQL`：说明 `.env.local` 未配置或 `direnv allow` 未执行。
- `relation "users" does not exist`：先执行 `make db-bootstrap-dev ...` 导入基线 SQL。
- `ModuleNotFoundError: glimmer`：先执行 `make deps`。

---

## 6. 常用命令速查

```bash
make help
make db-upgrade
make db-downgrade
make db-revision MSG=add_xxx
```

---

## 7. 启动 glimmer-app gRPC 服务（新增）

进入子包目录并启动：

```bash
cd /app/packages/glimmer-app
uv sync
uv run glimmer-app serve-grpc
```

指定监听地址/端口：

```bash
uv run glimmer-app serve-grpc --host 0.0.0.0 --port 50051
```

指定最大并发 RPC：

```bash
uv run glimmer-app serve-grpc --max-workers 200
```

停止方式：前台运行时按 `Ctrl + C`，服务会执行优雅关闭。
