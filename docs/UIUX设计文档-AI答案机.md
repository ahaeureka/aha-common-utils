## 1. 设计目标

### 1.1 产品设计目标

照见一念（Glimmer）不是传统聊天产品，也不是娱乐型测试产品，而是一个围绕“启发、反思、行动”构建的轻量心理工具。因此 UI/UX 设计需要同时满足以下目标：

1. **降低进入门槛**
   - 用户应在极短时间内进入状态，不需要学习复杂操作。
2. **承接情绪**
   - 设计要能容纳焦虑、犹豫、脆弱、困惑等状态，而不是制造压迫感。
3. **建立轻仪式感**
   - 通过输入、抽取、翻卡、停顿、出现答案等过程，形成“被照见”的体验。
4. **引导反思而非灌输结论**
   - 页面结构和文案都应留白，帮助用户自己生成意义。
5. **推动低阻力行动**
   - 结果页不能停留在“有感觉”，而要自然过渡到一个很小的行动。
6. **兼顾传播与长期使用**
   - 单次结果应适合截图传播；长期使用路径应适合回看与自我积累。
7. **支持轻量沉淀与温和回访**
   - 历史页不只是保存结果，而应帮助用户低负担地看到自己的情绪线索和问题变化。

### 1.2 用户体验目标

用户在一次完整使用后，应感受到：

- 我被理解了
- 我没有被评判
- 我得到了一点新的视角
- 我不需要现在就做大决定
- 我至少知道下一步能做什么
- 我可以稍后回来继续，而不是这次没做完就算失败

### 1.3 设计关键词

建议统一以下设计关键词：

- 克制
- 温和
- 清明
- 留白
- 微光
- 东方感
- 不说教
- 有洞察但不强势

## 2. 设计原则

## 2.1 核心原则

### 原则 1：先安放，再引导

用户进入产品时常处于心理张力中。界面应先提供安全感，再逐步引导进入反思。

### 原则 2：一步只做一件事

每个页面只承载一个明确任务：

- 提问
- 识别情绪
- 接收启发
- 选一个视角
- 回答一个问题
- 带走一个行动

### 原则 3：减少“工具感”，增强“体验感”

避免做成表单或问卷。通过卡片、层次、动效与节奏，让用户感受到一种柔和的引导过程。

### 原则 4：避免强控制和高压 CTA

按钮和引导不使用命令式措辞，如“必须继续”“马上行动”。改用温和表达，如：

- 看看另一个角度
- 继续往下想一点
- 带走一个小行动

### 原则 5：视觉上给出“轻重量”

产品涉及情绪与人生问题，但界面不能沉重到让人退缩。要在深度感与可亲近之间取得平衡。

### 原则 6：保持结构化一致性

虽然内容生成是 AI 驱动，但 UI 必须高度结构化、稳定，让用户感到可预期、可信赖。

### 原则 7：商业化不能破坏心理安全感

如果后续引入广告或品牌合作内容，设计上必须遵守“先完成核心体验，再出现商业化元素”的顺序。

要求：

- 广告不能打断提问、生成、反思输入这些高专注节点
- 广告样式应与内容结构兼容，但必须明确标识“广告 / 品牌合作”
- 会员去广告预期需要在体验上保持一致，不能出现误触或伪装内容
- 任何广告位都应有稳定占位，避免内容跳动破坏阅读节奏

## 3. 用户场景与体验策略

## 3.1 高频使用场景

### 场景 A：深夜情绪困惑

用户特点：

- 情绪敏感
- 容易被文字语气影响
- 需要被接住，而不是被分析

设计策略：

- 默认进入 `照见模式`
- 视觉更安静、动效更柔和
- 优先展示启发句与情绪卡

### 场景 B：白天职业决策

用户特点：

- 想快速看清问题
- 更愿意接受结构化视角
- 对时间效率敏感

设计策略：

- 可推荐 `明断模式`
- 用更明确的信息层级
- 行动建议要更具体

### 场景 C：日常轻量抽问

用户特点：

- 不是强问题驱动
- 更偏内容消费与轻仪式

设计策略：

- 首页保留一句启发感文案
- 输入入口足够轻
- 支持快速完成一轮

## 4. 设计语言系统

## 4.1 品牌气质建议

中文品牌气质：

- 照见
- 一念
- 明心
- 问心

英文气质参考：

- Glimmer
- A gentle glimpse inward

品牌感受应接近：

> 在混沌里，有一点微光；在犹豫里，有一点清明。

---

## 4.2 视觉风格方向

### 方向 A：东方微光

适合当前产品定位，建议优先采用。

视觉特征：

- 深色底或柔和低饱和背景
- 带有晨雾、月光、薄雾、纸面、光晕感
- 高留白
- 较少强边框
- 模块以卡片和层叠光感区分

关键词：

- 静
- 深
- 柔
- 透
- 轻微神秘感

### 方向 B：现代心理工具

更偏国际化产品感。

视觉特征：

- 中性浅底或深浅双主题
- 清晰的卡片层级
- 更强的信息结构
- 适合后续数据分析页延展

关键词：

- 理性
- 清楚
- 可信
- 工具化

### 建议

MVP 采用 **东方微光 + 现代结构** 的混合路线：

- 视觉情绪偏东方微光
- 信息组织偏现代工具

## 4.3 色彩系统建议

## 主色建议

### 主背景色

- 夜雾黑：`#0F1115`
- 青灰深夜：`#151A20`
- 月白浅底（浅色模式备选）：`#F6F4EF`

### 品牌强调色

- 微光金：`#C8A96B`
- 雾紫：`#8C7AAE`
- 静海蓝：`#5D7C99`

### 辅助状态色

- 成功/行动：`#6FA58A`
- 警示/需观察：`#C28A52`
- 风险提醒：`#B56E6E`
- 中性文字灰：`#9AA3AD`

## 配色原则

- 不使用高饱和荧光色
- 强调色只用于关键按钮、标签和焦点
- 风险色不应制造惊吓感，避免纯亮红
- 大面积底色以沉静为主，形成“容纳感”

## 4.4 字体建议

### 中文

- 正文：思源黑体 / HarmonyOS Sans / 苹方
- 标题：思源宋体或衬线字体可小范围点缀，用于品牌感强化

### 英文

- Inter / SF Pro / Manrope

### 字体策略

- UI 信息用无衬线，保证可读性
- 品牌页标题或引导句可局部用衬线，增强气质
- 不建议大面积装饰性书法体

---

## 4.5 图形与插画建议

### 图形语言

- 柔和光斑
- 渐隐渐现的发光点
- 朦胧边界的卡片阴影
- 水波、圆环、雾面过渡

### 不建议

- 宗教符号过重
- 算命、塔罗、占卜符号过多
- 玄学、夸张宇宙星盘风格

原因：

产品是心理机制驱动，不是玄学工具。

- 结果页和行动页保留“再问一次”

## 6. 核心页面设计规范

## 6.1 页面 1：首页 / 提问页 `AskPage`

### 页面目标

让用户毫无负担地提出问题。

### 信息层级

1. 品牌名 / slogan
2. 一句引导文案
3. 输入框
4. 示例问题
5. 主按钮
6. 次级入口：模式选择（可选）

### 建议文案

标题：

- 问一个此刻最想知道的问题

副标题：

- 不急着得到答案，先让问题慢一点被看见。

### 组件说明

#### 输入框

- 占主要视觉区域
- 多行输入
- 默认 placeholder 应真实、温和

placeholder 示例：

- 我要不要离开现在的工作？
- 我是不是该放下了？
- 为什么我总是下不了决心？

#### 示例问题

- 用 Chip 或轻量按钮呈现
- 点击可自动填充，不直接提交

#### 主按钮

文案建议：

- 开始照见
- 问问看
- 看一个角度

避免：

- 立即分析
- 立刻获取答案

### 交互说明

- 聚焦输入框时，背景可轻微暗化，帮助用户专注
- 提交后进入短暂过渡页，形成仪式感
- 可提供次级入口“先缓一缓”与“我还说不清”

### 状态

- 默认态
- 输入中
- 不可提交态
- 提交中 Loading 态
- 错误态

---

## 6.2 页面 2：情绪识别层 `EmotionGate`

### 页面目标

在不增加负担的前提下，帮助用户命名当前状态。

### 设计策略

- 视觉轻于正式问卷
- 采用选择标签而非下拉或表单
- 支持“跳过”

### 信息结构

标题：

- 你现在更接近哪种状态？

副文案：

- 这会帮助我更贴近你的问题。

选项展示：

- 焦虑
- 很累
- 舍不得
- 不甘心
- 期待又害怕
- 说不清

### 按钮

- 主操作：继续
- 次操作：先跳过

### 交互细节

- 选中标签有柔和描边和背景变化
- 不使用强对比高亮
- 支持一步完成

### 衔接要求

- 若用户明显处于高情绪张力中，可先进入稳定入口，再返回此页

## 6.2A 页面 2A：稳定入口 / 先缓一缓页 `SoothingGate`

### 页面目标

让用户在还无法组织语言时，先获得 20–60 秒的心理缓冲，再决定是否进入正式提问和反思。

### 页面结构

1. 一句容纳式标题
2. 呼吸或节律引导区
3. 简短稳定文案
4. 继续提问 / 先随便说说 / 稍后再来

### 建议文案

标题：

- 如果你现在很乱，先不用急着想清楚

副标题：

- 先让这一口气慢一点，再看看要不要继续。

### 设计要求

- 页面应比正式流程更安静、更留白
- 用柔和的圆环、呼吸点、微光层替代复杂动效
- 不营造冥想 App 或睡眠 App 的强功能感

### 交互要求

- 默认提供 30 秒版本
- 用户可一键跳过
- 结束后可进入提问页或自由倾诉页

### 低保真线框

```text
+------------------------------------------------------+
| 返回 | 照见一念                               稳定中 |
|------------------------------------------------------|
|                                                      |
|        如果你现在很乱，先不用急着想清楚             |
|        先让这一口气慢一点，再看看要不要继续         |
|                                                      |
|                    ( 呼吸光环动画 )                  |
|                                                      |
|              [ 30秒 ] [ 60秒 ] [ 跳过 ]             |
|                                                      |
|         [ 继续提问 ] [ 先随便说说 ] [ 稍后再来 ]     |
|                                                      |
+------------------------------------------------------+
```

### 组件清单

- `TopProgressHeader`
- `SoothingTitleBlock`
- `BreathingHalo`
- `DurationSegmentedControl`
- `SecondaryTextHint`
- `PrimaryActionButton`
- `GhostActionButton`
- `LaterReturnButton`

### 关键交互状态

- 默认态：展示 30 秒默认方案与三个去向按钮
- 呼吸进行态：中央节律动画启动，按钮降噪，仅保留跳过与结束后去向
- 已完成态：弱化动画，突出“继续提问”“先随便说说”
- 跳过态：保留最小过渡，避免像失败退出
- 弱网 / 加载态：允许先进入下一步，异步补记稳定行为埋点

## 6.2B 页面 2B：自由倾诉页 `UnloadPage`

### 页面目标

允许用户先说碎片、抱怨或感受，再由系统帮助整理成可进入主链路的问题。

### 页面结构

1. 容纳式开场文案
2. 文本输入或语音转写区
3. AI 提炼结果卡
4. 继续提问 / 重新整理

### 辅助文案示例

- 不需要说完整，你可以先把脑子里的那一团说出来。
- 如果你现在还不知道怎么问，就先随便说说。

### 设计要求

- 视觉上比正式提问更松弛
- 输入区域像倾诉卡，不像客服对话框
- 提炼结果卡要明确体现“这是帮你整理出来的问题”

### 低保真线框

```text
+------------------------------------------------------+
| 返回 | 照见一念                               整理中 |
|------------------------------------------------------|
| 不需要说完整，你可以先把脑子里的那一团说出来。     |
|                                                      |
| +--------------------------------------------------+ |
| | 文本输入 / 语音转写区域                           | |
| | 我最近一直很累，我不知道是不是该离职……           | |
| +--------------------------------------------------+ |
|                                                      |
| [ 语音输入 ] [ 清空 ] [ 帮我整理成问题 ]            |
|                                                      |
| +--------------------------------------------------+ |
| | 帮你整理出来的问题                                | |
| | 我要不要因为持续疲惫而重新评估当前工作？         | |
| +--------------------------------------------------+ |
|                                                      |
|             [ 就从这个问题开始 ] [ 再整理一下 ]     |
+------------------------------------------------------+
```

### 组件清单

- `UnloadIntroText`
- `FreeformTextArea`
- `VoiceTranscribeButton`
- `TranscribeStateChip`
- `RefineQuestionButton`
- `RefinedQuestionCard`
- `StartFromQuestionButton`
- `RetryRefineButton`

### 页面级模块清单

1. 顶部进度与退出区
2. 容纳式开场文案区
3. 自由输入区
4. 语音转写入口与转写状态区
5. 提炼操作区
6. 提炼结果预览区
7. 继续 / 重整动作区

### 关键交互状态

- 空白态：输入区占主视觉，结果卡隐藏
- 输入中态：显示字数提示与“帮我整理成问题”可用态
- 语音录制态：输入区切为转写反馈，强调停止录音
- 转写处理中：按钮禁用，展示轻量处理中状态
- 提炼完成态：展示主问题结果卡与二次操作按钮
- 提炼失败态：保留用户原文，提示重试而非清空
- 离开确认态：若已有内容未整理，提示保存草稿或稍后继续

## 6.2C 页面 2C：问题整理确认页 `QuestionRefinePage`

### 页面目标

在正式进入 Trigger 之前，让用户确认“这是不是我现在真正想问的问题”，并选择一个更适合继续的侧重点。

### 页面结构

1. 上一步碎片表达摘要
2. 系统提炼出的主问题卡
3. 1–2 个可选侧重点
4. 去向选择区：继续反思 / 先缓一缓 / 先不决定
5. 返回修改入口

### 设计要求

- 视觉上应比自由倾诉页更收束，体现“混乱已经被整理了一点”
- 主问题卡必须是页面焦点，让用户一眼判断是否被整理准确
- 侧重点应呈现为轻量选择，而不是复杂分支树
- 去向选择区应清楚表达“你仍然可以慢一点，不必马上进入答案”

### 页面级模块清单

1. `UnloadSummaryPreview`
2. `PrimaryQuestionCard`
3. `FocusOptionCardList`
4. `RefineConfidenceHint`
5. `ContinueToTriggerButton`
6. `GoToSoothingButton`
7. `DecideLaterButton`
8. `BackToEditLink`

### 关键交互状态

- 默认确认态：主问题卡高亮，展示 1–2 个侧重点
- 切换侧重点态：实时切换后续引导文案，不刷新整页
- 确认进入态：点击“就从这个问题开始”后进入生成中间态
- 改写返回态：允许回到 `UnloadPage` 修改原表达，不丢失内容

- 先不决定态：弹出保存说明，用户可仅保存提炼结果并退出

## 6.7 页面 7：总结与微实验页 `ActionPage`

### 页面目标

帮助用户把模糊感受转为一个更具体的理解和下一步。

### 页面结构

1. AI 总结卡
2. 认知偏差提醒（如命中）
3. 未来自我模块（如命中）
4. 微实验卡
5. 行动标签
6. 如果—那么计划

### 模块 3：未来自我

- 小行动
- 为什么是这一步
- 标签
- 如果—那么计划（可选）

- 低风险
- 可逆
- 信息收集
- 自我澄清

- 我带走这个行动
- 保存这次照见
- 再问一个问题
- 晚点再回来看看

### 商业化位预留建议

若该页后续承接商业化，建议仅在主内容阅读完成后，于页面底部预留 1 个低打扰原生位或专题合作卡位。

设计要求：

- 默认位于总结内容之后，而不是行动按钮之前
- 高度稳定，避免异步加载后整体跳动
- 使用独立容器，与 AI 内容卡视觉上可区分
- 明确显示“品牌合作”或“推荐内容”标签
- 该位点需要预留曝光、点击、关闭三类埋点触发点

1. 答案卡

### 不建议

- 柔和呼吸点

## 8. 动效与节奏设计

## 8.1 动效目标

动效不是装饰，而是服务于心理节奏：

- 输入之后有停顿
- 停顿之后才出现结果
- 卡片切换带一点“翻开”的感觉
- 行动建议出现时有“落地”的感觉

## 8.2 动效原则

- 总体缓、轻、短
- 避免炫技和过强位移动效
- 优先使用透明度、轻缩放、轻上浮

## 8.3 建议动效

- 页面进入：淡入 + 上浮 8px
- 卡片出现：Stagger 依次淡入
- 结果出现：答案先出，解释后出
- 选卡：轻微缩放 + 高亮边框
- 保存成功：微弱勾选反馈

### 时长建议

- 微交互：$120	ext{ms} 	ext{–} 180	ext{ms}$
- 卡片切换：$220	ext{ms} 	ext{–} 280	ext{ms}$
- 页面淡入：$240	ext{ms} 	ext{–} 320	ext{ms}$

---

## 9. 文案与语气规范

## 9.1 品牌语气

应统一为：

- 温和
- 清晰
- 克制
- 有理解力
- 有一点诗性

## 9.2 避免语气

- 指责型：你就是太……
- 命令型：你必须……
- 神谕型：命运告诉你……
- 机械型：已为你生成结果

## 9.3 文案场景示例

### 好的表达

- 也许你可以从这里想下去
- 先别急着为今天下结论
- 你现在更需要的是看清，而不是马上决定

### 不好的表达

- 你的问题原因已经很明显
- 你必须停止这样做
- 正确答案是

---

## 10. 可访问性与包容性设计

## 10.1 可访问性要求

- 文本对比度满足基础可读要求
- 关键按钮点击面积不小于移动端标准
- 标签与按钮不可仅靠颜色区分
- 重要反馈需兼顾文字与视觉提示

## 10.2 情绪脆弱场景考虑

- 不使用突然闪烁和强刺激动效
- 不使用高压红色提醒
- 风险内容需温和中断常规流程

## 10.3 安全提示设计

当检测到高风险内容时，页面需切换为安全模板：

- 弱化品牌氛围感
- 强化清晰、直接、支持型信息
- 提供明确资源和下一步

## 10.4 广告位与埋点的 UX 预留原则

关联文档：

- [广告商业化产品方案与页面规划.md](广告商业化产品方案与页面规划.md)
- [广告位配置与广告接入实施步骤.md](广告位配置与广告接入实施步骤.md)
- [广告位配置与广告接入研发Checklist.md](广告位配置与广告接入研发Checklist.md)

为了避免后期临时插入广告破坏体验，设计阶段就应明确哪些页面允许出现商业化位，以及需要保留哪些埋点触点。

### 推荐预留页面

- 启发结果页底部
- 总结与微实验页底部
- 历史记录页信息流中
- 每日卡牌页品牌合作区

### 不建议预留页面

- 首页提问主区
- Loading 页
- 深入回答输入区
- 风险提示页

### 每个广告位至少预留的埋点触点

- 曝光：广告位达到有效可视条件
- 点击：用户点击广告主体或 CTA
- 关闭：用户主动收起或隐藏广告
- 激励完成：用户完整看完激励内容并成功解锁

### 设计交付时应补充给前端的字段

- `slotId`
- `placement`
- `slotType`
- `labelType`（广告 / 品牌合作 / 推荐）
- `closable`
- `rewardType`（如有激励式广告）

## 13. 前后端协作要点

## 13.1 前端实现注意事项

- 结果结构固定，不依赖原始大段 AI 文本直出
- 各模块需支持条件渲染：
  - 无情绪识别
  - 无偏差提醒
  - 无未来自我
  - 无 `ifThenPlan`
- 长文本与短文本都需有兼容样式

## 13.2 后端与 AI 输出约束

设计依赖以下结构稳定输出：

- `answerText`
- `hintText`
- `cards[]`
- `reflectionSummary`
- `cognitiveBiases[]`
- `futureSelfMessage`
- `action.tags[]`
- `ifThenPlan`

若字段缺失，需有兜底展示逻辑。

## 14. UX 验证建议

## 14.1 首轮可用性测试重点

建议优先验证：

1. 用户是否能在首页 3 秒内理解产品用途
2. 用户是否愿意输入真实问题
3. 用户是否理解“照见模式 / 明断模式”的区别
4. 用户是否愿意继续点击卡片
5. 用户是否觉得微实验“真能做”
6. 用户是否愿意使用“先缓一缓”或“先随便说说”入口
7. 用户是否理解历史页是“轻量情绪日记”而不是聊天记录

## 14.2 定性访谈建议问题

- 你觉得这个产品更像什么？
- 哪个页面最让你有“被理解”的感觉？
- 哪一步让你最想退出？
- 哪种文案会让你觉得被评判？
- 你会愿意保存或分享哪一页？

## 14.3 定量验证指标

- 首页到结果页转化率
- 结果页到卡片页转化率
- 卡片点击率
- 回答完成率
- 微实验采纳率
- 保存率
- 分享率
- 情绪日记补记率
- 历史页回访率

## 16. 一句话总结
